# Graph-Augmented RAG Subsystem Architecture

This document describes the workflow of a user query through the graph-augmented Retrieval-Augmented Generation (RAG) subsystem.

```
User Query
    │
    ▼
Hybrid Retrieval
    │
    ▼
Knowledge Graph Expansion
    │
    ▼
Context Assembly
    │
    ▼
Prompt Construction
    │
    ▼
LLM Synthesis
    │
    ▼
  Answer
```

## Flow Description

1. **User Query**: Incoming text questions submitted by requesting client interfaces.
2. **Hybrid Retrieval**: Extracts top-k text candidate chunks using semantic search (Qdrant) and lexical keyword matching (BM25).
3. **Knowledge Graph Expansion**: Translates query terms and chunk identifiers into seed entity coordinates, then queries the `GraphContextExpander` to traverse Neo4j relationships.
4. **Context Assembly**: Consolidates textual matching candidate chunks and neighboring entity graph nodes into a unified `AssembledContext` model.
5. **Prompt Construction**: Renders the complete system instructions and context-injected user query block through the `PromptBuilder`.
6. **LLM Synthesis**: Submits the final prompt schema model to configured LLM synthesis clients (`SynthesisService`).
7. **Answer**: The response string generated by the LLM is packaged inside `RagResponse` and returned downstream.
